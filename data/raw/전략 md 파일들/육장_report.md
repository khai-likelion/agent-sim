# 🏮 [X-report] 육장 생존 전략 처방전

## 🚨 1. 매장 핵심 스코어 (Scoreboard)
- **종합 등급: A**  
  - 이유: **맛(0.90)**·**청결(0.80)**이 상권 기준 대비 확실한 우위. 다만 **가성비(0.60)**·**서비스 일관성(불친절/편차)**이 재방문과 객단가 방어의 핵심 리스크라 “A(강점 뚜렷, 운영 개선 시 상위 고착 가능)”로 판단.

- **항목별 성적표** *(10칸=1.0 만점 환산)*  
  - [맛] **▮▮▮▮▮▮▮▮▮▯** (**0.90** / 상권 평균 추정 **0.791**)  
  - [서비스] **▮▮▮▮▮▮▮▯▯▯** (**0.70** / 상권 평균 추정 **0.68**)  
  - [가성비] **▮▮▮▮▮▮▯▯▯▯** (**0.60** / 상권 평균 추정 **0.58**)  
  - [청결도] **▮▮▮▮▮▮▮▮▯▯** (**0.80** / 상권 평균 추정 **0.72**)  

- **객단가 진단:** **객단가 40,722원** → 상권 대비 **정보 부족(비교 데이터 미제공)**  
  - 다만 “비싸다” 인식 리뷰 + 가성비 점수 **0.60**을 감안하면 **체감상 ‘높게 느껴질 가능성’**이 큼.

## 📊 2. 데이터가 말하는 현재 (팩트 체크)
- **데이터 분석 요약**
  - 상권은 **최근 4년 점포 수 증가 + 개업률>폐업률(성장 국면)** 신호가 있음(연도 오타 데이터는 제거 처리).  
  - 매출은 **주말 집중**, 시간대는 **저녁 17–21 피크**.  
  - 유동인구는 **00–06 비중이 가장 큼(28.8%)**이나, 이는 외식 전환율이 낮은 심야 유동일 수 있어 **매출 피크와 불일치**(야간 유동=식사 수요로 직결 X 가설).
  - 리뷰 지표상 포지션은 “**맛으로 끌고 오지만, 가격 체감/품질 편차/서비스 편차가 발목**” 구조.

- **경쟁사 대비 강점 (데이터 기반 2가지)**
  1) **맛: 0.90(평균 0.791 대비 우위)** → 핵심 자산(대체 불가 영역)  
  2) **청결: 0.80(평균 0.72 대비 우위)** → 고기/한식에서 신뢰를 만드는 기반

- **경쟁사 대비 약점/리스크 (데이터 기반 2가지)**
  1) **가성비 절대값 낮음: 0.60** + “비싸다” 인식 → 재방문 빈도/전환율에 직접 타격  
  2) **갈비 품질/양 편차(질김·양 적음) + 불친절 케이스** → 강점(맛)을 훼손하는 치명 타입 불만

## 🧭 3. 승부를 결정지을 '선택형 필살기' (Simulation Candidates)
※ 아래 제안된 솔루션 중 **시뮬레이션에서 테스트해 보고 싶은 항목**을 선택해 주세요.  
※ 선택된 솔루션은 디지털 트윈 상권 내 3,800명 에이전트의 방문 확률에 직접적인 영향을 줍니다.

### 🔹 전략 카테고리 1: [제품 매력]
- **솔루션 A: 갈비 ‘중량/컷/숙성/굽기’ 표준화(레시피 카드 + 계량)**
  - 변수화 포인트: 1인분 **중량(g)**, 숙성/해동 방식, 굽기 시간, 제공 온도  
  - 기대 효과: “질김/양 적음” 리뷰 감소 → **맛 강점(0.90) 방어 + 재방문 상승**

- **솔루션 B: ‘가성비 체감’ 메뉴 구조 재설계(가격 인하 없이)**
  - 예: 기본 구성에 **공깃밥/사리 옵션 정책 명확화**, 세트에 “포함”을 박아 체감 개선  
  - 변수화 포인트: 기본 포함 품목(밥/사리/반찬) 유무, 세트 구성  
  - 기대 효과: “비싼데 양도 적다” 프레임 완화 → **가성비 지표(0.60) 개선**

- **솔루션 C: 피크타임(주말·17–21) 전용 ‘품질 보호’ 운영(한정 수량/프리프)**
  - 예: 주말 저녁엔 특정 메뉴 **한정 수량** + 미리 준비(프리프)로 품질 흔들림 방지  
  - 변수화 포인트: 한정 수량 도입 여부, 프리프 비율  
  - 기대 효과: 피크에 발생하는 품질 편차를 줄여 **리뷰 변동성 감소**

---

### 🔹 전략 카테고리 2: [재방문 락인]
- **솔루션 A: 재방문 트리거 설계(스탬프/쿠폰) — ‘2회차 장벽’ 제거**
  - 예: “2번째 방문 시 **된장/냉면/사리 중 1개 제공**”처럼 체감 큰 보상  
  - 변수화 포인트: 리워드 종류, 발급 조건(당일/7일/14일 내)  
  - 기대 효과: “만족은 높지만 자주 못 감”을 **재방문 행동으로 전환**

- **솔루션 B: 리뷰 불만 상쇄형 ‘즉시 보상’ 룰(컴플레인 처리 표준)**
  - 예: 질김/양/응대 불만 제기 시 “즉시 교체/추가 제공”을 직원 권한으로 부여  
  - 변수화 포인트: 교환/추가 제공 허용 범위, 처리 시간 기준  
  - 기대 효과: 부정 리뷰를 **현장에서 차단** → 평점 방어 + 이탈 감소

- **솔루션 C: 객단가 방어용 ‘업셀’이 아닌 ‘납득형 추가’ 설계**
  - 예: 추가 메뉴를 “가성비”로 보이게(소용량/합리 가격) 구성해 거부감 최소화  
  - 변수화 포인트: 추가 메뉴 가격대/구성(소용량/공유형)  
  - 기대 효과: 가격 민감 고객에게도 **추가 구매 허들↓**, 객단가 안정화

---

### 🔹 전략 카테고리 3: [공간 경험]
- **솔루션 A: 피크타임 서비스 흔들림 방지(동선/역할 분담 + 응대 스크립트)**
  - 예: 주말·저녁에 “입장/주문/그릴/계산” 역할 고정 + 3문장 응대 표준  
  - 변수화 포인트: 피크타임 인력 배치(1명 추가 여부), 역할 분담 방식  
  - 기대 효과: “불친절” 같은 치명 리뷰 감소 → **서비스(0.70) 안정화**

- **솔루션 B: ‘기다림 체감’ 최소화(웨이팅 안내/회전 개선 장치)**
  - 예: 웨이팅 시 예상 대기시간 고지, 주문 선입력, 테이블 세팅 프리셋  
  - 변수화 포인트: 웨이팅 안내 도입 여부, 주문 선입력 여부  
  - 기대 효과: 회전(turnover **0.50**)이 동급인 상황에서 **체감 만족을 올려 이탈 감소**

- **솔루션 C: 청결 강점을 “보이게” 만드는 체크리스트 공개(신뢰 강화)**
  - 예: 테이블/그릴/화장실 점검 시간표를 고객이 보이는 곳에 간단히 게시  
  - 변수화 포인트: 공개 여부, 점검 주기  
  - 기대 효과: 청결 우위(**0.80**)를 인지 자산으로 전환 → 첫 방문 전환율 개선
